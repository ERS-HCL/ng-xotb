Introduction

The colour system for a product has many essentials and restraints. There is a need to be intentional and functional with colour use. While xOTB provides the reference colours, you may, customize it at extreme levels to suite your product needs but respecting design values is the core of any design system.

Design Values

  • Purposeful: use colours in a functional way to show intent when needed.
  • Accessible: use colours with as much contrast as possible while still being beautiful. Avoid muddy neutral colours.
  • High Dynamic Range: use very light neutral colours for background colours. Allow for a wide range of the colour spectrum.

Implementation Details

The colour system in xOTB is located in the theme and is used throughout the theme. There is no real dependency on any of the colours directly within components. Components always access a theme colour or property through `variable`. For example, $text-color is a variable name for default text color in the Verve theme.

Examples

    
      {{
        'body {
            background-color:$background-color
            color: $text-color
        }'
      }}
